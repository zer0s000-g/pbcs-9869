## The compliance checklist mindset

PBCS compliance isn't a form you submit once. It's a **continuous state** you maintain:

```
Before operations:       Show capability → Get approval
During operations:       File correctly → Operate to spec
After operations:        Collect data → Analyze → Report
If performance degrades: Identify → Investigate → Correct → Re-prove
```

---

## Common compliance gaps

> [!warning] Gap 1: Assuming equipment = compliance
> Having CPDLC/ADS-C avionics installed is necessary but not sufficient. You also need procedures, training, MEL provisions, service provider agreements, and monitoring arrangements.

> [!warning] Gap 2: Skipping monitoring after approval
> Many operators treat approval as the finish line. It's actually the start line — post-implementation monitoring is mandatory under Doc 9869 Chapter 4.5.

> [!warning] Gap 3: Not coordinating with service providers
> Your CSP/SSP needs to provide outage data, performance reports, and support investigations. If you haven't arranged this contractually before approval, you'll have a compliance gap on day one.

## Compliance posture

Doc 9869 guidance must be combined with:
- Current State/regional procedures (regulations, AIP, SUPP)
- Company-specific approval evidence
- Active service provider arrangements
- Operational monitoring data

This KB provides the source routing — the actual compliance artifacts are yours to create and maintain.
