# APAC PBCS Regulatory Mapping

> [!summary] TL;DR
> How Doc 9869 translates to actual PBCS requirements in the Asia/Pacific region. This page maps the region's implementation programmes, State prescriptions, and monitoring arrangements.

---

## APAC regional framework

The ICAO Asia/Pacific (APAC) region has one of the world's most active PBCS implementation programmes through the **FIT-ASIA** initiative and individual State prescriptions.

| Layer | What it covers | Reference |
|---|---|---|
| **Regional plan** | APAC Seamless ATM Plan, FIT-ASIA phases | APAC ANP (Air Navigation Plan) |
| **Regional SUPPs** | Doc 7030 — APAC-specific PBCS procedures | Doc 7030/APAC |
| **State AIPs** | Individual State PBCS requirements per FIR | Each State's AIP ENR section |
| **Monitoring** | APAC PBCS Task Force monitoring programme | Regional monitoring reports |

## FIT-ASIA implementation phases

FIT-ASIA (Future Air Navigation Systems Interoperability Team — Asia) coordinates PBCS deployment:

| Phase | Airspace | Specs | Target |
|---|---|---|---|
| **Phase 1** | Bay of Bengal, South China Sea | RCP 240/D, RSP 180/D | Completed |
| **Phase 2** | Pacific, Indian Ocean eastern | RCP 240/D, RSP 180/D | In progress |
| **Phase 3** | Additional oceanic/remote blocks | RCP/RSP per operational need | Planned |

## How to use this mapping

### If you operate in APAC airspace

1. Identify which airspace block(s) your route passes through (table above)
2. Check the prescribing State's AIP ENR section for exact requirements
3. Verify which RCP/RSP combinations are prescribed
4. Check if a PBCS monitoring programme is active for that airspace
5. Prepare your compliance evidence per [[PBCS Compliance Worked Example]]

### If you're an ANSP prescribing PBCS

1. Coordinate with the APAC PBCS Task Force on specification alignment
2. Publish RCP/RSP requirements in your AIP ENR section
3. Join or establish a regional monitoring programme
4. Use [[Developing RCP RSP Specifications]] and [[Applying RCP RSP Specifications]]

### If you're a State authority

1. Review APAC ANP and Doc 7030/APAC for regional obligations
2. Harmonize your prescriptions with neighbouring FIRs
3. Establish bilateral/multilateral approval recognition where possible
4. Participate in the APAC PBCS Task Force

## Typical APAC AIP ENR structure

When reviewing a State's AIP for PBCS requirements, look for:

```
AIP ENR 1.8 — Regional Supplementary Procedures
  ├── 1.8.x — PBCS requirements for [specific FIR]
  ├── RCP/RSP specifications prescribed
  ├── Aircraft/operator eligibility criteria
  ├── Service provider requirements
  └── Monitoring and reporting obligations
```

## Regional monitoring programme

The APAC PBCS Task Force coordinates a regional monitoring programme:

| Element | Description |
|---|---|
| **Coordinator** | APAC PBCS TF (rotating chair) |
| **Data sources** | Participating ANSPs submit monthly performance data |
| **Report format** | Standardized per Doc 9869 Appendices D/E |
| **Frequency** | Monthly data collection, quarterly regional reports |
| **Corrective action** | Coordinated through the TF; per-State implementation |

> [!note] Monitoring programme detail
> For the full monitoring framework, see [[Post-Implementation Monitoring]] and [[Monitoring Programme]]. This page focuses on the regional coordination layer.
