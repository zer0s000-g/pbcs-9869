# PBCS Implementation Risks

| Risk | Why it matters | Mitigation |
|---|---|---|
| Treating PBCS as only avionics approval | Misses ANSP, CSP/SSP, monitoring, and State responsibilities | Use [[PBCS Responsibility Matrix]] |
| Missing post-implementation monitoring | Initial compliance can degrade in real operations | Use [[Monitoring Programme]] |
| Poor source traceability | Colleagues cannot defend claims | Use [[Evidence and Traceability Map]] |
| Confusing RCP and RSP | Communication and surveillance criteria are not interchangeable | Use [[RCP vs RSP]] |
| Service-provider outage data gaps | Availability and corrective action become weak | Use [[Data Collection]] and service agreements |
