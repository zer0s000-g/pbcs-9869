# Post-Implementation Monitoring

## Source-hardened definition
Post-implementation monitoring is the continued data collection, analysis, problem reporting, tracking, and corrective-action process used to verify that real operations continue to meet the prescribed RCP/RSP performance.

## Operational pattern
1. Collect transaction, report, availability, and outage data.
2. Filter data so the analysis measures operationally relevant performance.
3. Compare observed ACP/ACTP/PORT/ASP and availability against applicable RCP/RSP criteria.
4. Analyze subsets by media, operator, aircraft type, airframe, service provider, and route/airspace where relevant.
5. Report locally/regionally as applicable.
6. Open, investigate, coordinate, and close problem reports.

## Company-use boundary
Monitoring is not optional follow-up; it is the evidence layer that detects performance drift after initial implementation. However, the exact reporting format and thresholds must be aligned with the applicable State/regional programme and the operation's prescribed RCP/RSP specification.
