# Monitoring Programme

## Summary
A monitoring programme collects data, evaluates actual performance against RCP/RSP criteria, identifies degradation, reports problems, and tracks corrective action.

## Source basis
- Doc 9869 treats post-implementation monitoring as necessary to confirm that actual communication and surveillance performance continues to meet the applicable RCP/RSP specifications.
- Appendix D supports CPDLC and ADS-C monitoring; Appendix E supports SATVOICE monitoring. Table-level checking is tracked in [[Appendix D and E Monitoring Table Verification]].
- For report structure, see [[Monitoring Report Template]].

## Practical outputs
- Named owner.
- Required evidence.
- Monitoring data source.
- Review cadence.
- Escalation/corrective-action path.
